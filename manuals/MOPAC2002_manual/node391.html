M.O.

This keyword is to be used only when MANUAL is also specified. A molecular orbital to be used as source of density matrix. The M.O. comes after the other data required by MANUAL as a set of free-format coefficients. After all these data are read in, the program continues as usual, but many limitations apply. For example, if M.O. was used, then the key-words PSI=n, HOMO and LUMO would all give the same result. These limitations are due to the absence of other data. The second set of key-words defines the type of plot to be drawn; much of this data is obligatory in that there are no defaults for certain data. The user must supply:
  • A definition of the center of the plot.
  • A definition for the axis of the vector perpendicular to the plot.
  • A length, in Ångstroms, of the edge of the plot.
Other data, which are mutually exclusive, are:
  • A molecular orbital number, given by PSI=n, or
  • A highest-occupied molecular orbital, defined by the key-word HOMO, or
  • A lowest-unoccupied molecular orbital, defined by the key-word LUMO, or
  • The explicit molecular-orbital occupancy, see key-word OCCUPANCY, or
  • The normal electron density of the molecule. This is specified by the absence of the other key-words; i.e., this is the default.
If a full electron density map is specified. (That is, if the last option above is taken), then another key-word, BONDS, can be used. BONDS will remove the electron density due to the atom from the plot. The effect of this is to generate a plot showing where the charge build-up in the bonds occurs. Note: Ionic separations are not shown by BONDS.
