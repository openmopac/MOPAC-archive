The shut command

If, for whatever reason, a run needs to be stopped prematurely, the command shut <jobname> can be issued. This will execute a small command-language file, which copies the data-file to form a new file called <filename>.end.

The next time MOPAC calls function SECOND, the presence of a readable file called <filename>.end, is checked for, and if it exists, the apparent elapsed CPU time is increased by 10,000,000 seconds, and a warning message issued. No further action is taken until the elapsed time is checked to see if enough time remains to do another cycle. Since an apparently very long time has been used, there is not enough time left to do another cycle, and the restart files are generated and the run stopped.

The shut command is completely machine-independent.
