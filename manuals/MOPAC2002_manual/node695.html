Overview

MOPAC is distributed in compressed form. The top copy is designed for use with the following scalar compilers:

1. Pentium II/Pro running Linux with Portland Group compiler.
2. The same, but using GNU g77 compiler.
3. DEC Alpha workstation with UNIX 4.0 and Fortran 5.0.
4. SGI R5000 Indy running IRIX 6.2 and Fortran 7.1.
5. SGI R10000 PowerChallenge running IRIX 6.2 and Fortran 7.2.
6. Sun UltraSPARC with Fortran 4.2.
7. Generic Sun SPARC with Fortran 4.2.
8. Older Suns with Sun 2.0.1 compiler.
9. HAL workstations with Fujitsu compiler.
10. IBM workstations.
11. Fujitsu VX/VPP300/VPP700 series.

and parallel compilers:

1. Fujitsu AP3000 using Sun Fortran 77 compiler.
2. Fujitsu AP3000 using Fujitsu Fortran 90 compiler.
3. Fujitsu VX/VPP300/VPP700 running MPI 1.1.
4. Multiprocessor SGI PowerChallenge running SGI's MPI.
5. Sun Ultra Enterprise 10000 running MPICH.

To get MOPAC up and running, carry out the following instructions:

1. Obtain a copy of MOPAC 2002, by E-mail to mwada@se.fujitsu.co.jp.
2. Create a directory to hold MOPAC (mkdir mopac).
3. Move into the MOPAC directory (cd mopac).
4. Copy the distribution file into this directory.
5. If the file is compressed, uncompress it. Compressed files that end in ".Z" should be uncompressed with uncompress, files that end in ".gz" should be gunzip'd.
6. The distribution file is stored as UNIX "tar" form, so read the file using tar -xvf <file>
7. Go into the ./src directory.
8. Before the Makefile can be run, it needs to know what set of compiler options to use. These sets of options are stored in files in the sub-directory MAKE_INCLUDE. Select the appropriate set and note the name of the file.
9. Make this name known to the Makefile by issuing the command: 
setenv MOPAC_PLATFORM <file>
in which <file> is the name of the file, after removing the prefix "md.". For example, to select the options in the file md.Sun_SPARC, issue the command: 
setenv MOPAC_PLATFORM Sun\_SPARC
10. Run the Makefile by issuing the command make.
