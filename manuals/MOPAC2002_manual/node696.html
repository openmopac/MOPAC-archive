sizes.h

The file sizes.h determines how big a system MOPAC can run. The current maximum size of MOPAC, in terms of atoms, can be found in the output file near the start, at the end of the fourth line of stars. The end of that line would look something like this:

******************020000

This means that the program was configured to run systems of up to 20,000 atoms. Whenever you modify sizes.h, re-run make.

When make is run, it compiles all the FORTRAN files, using the array sizes given in sizes.h : these should be modified before make is run. If, for whatever reason, sizes.h needs to be changed, then make should be re-run, as modules compiled with different sizes.h will be incompatible.

   The parameters within sizes.h that the user can modify are NUMATM, MAXTIM and MAXDMP.  NUMATM is assigned a value equal to the largest number of atoms that a MOPAC job is expected to run. Note, however, that because MOPAC now uses dynamic memory allocation, the size of the program depends mainly on the size of the system being run, and only to a very small degree on the value of NUMATM. Because of this, the value of NUMATM can be quite large, for example, 100-20,000.

MAXTIM is the default maximum time in seconds a job is allowed to run before either completion or a restart file being written. MAXDMP is the default time in seconds for the automatic writing of the restart files. If your computer is very reliable, and disk space is at a premium, you might want to set MAXDMP as MAXDMP=999999.

If you want, MESP can be varied.

The quantity ISAFE is used to decide whether memory or reliability is the more important. If ISAFE is set to 1, then a larger amount of memory will be needed, but the reliability of MOPAC is increased. If the special SCF convergers are needed, and ISAFE=1, then they will be used. If ISAFE=0, then MOPAC will not allocate memory for these special SCF convergers. Whichever choice is made, the other choice can be selected at run-time by use of keywords. Thus if ISAFE=0 is selected, then the run-time default is to minimize memory usage. If keyword SAFE is used, then the other option will be used for that job. See also keywords SAFE and UNSAFE
