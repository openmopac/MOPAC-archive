RELTHR

When RELTHR=n is present in a MOZYME calculation, the default value for deciding whether an atom should be present in a LMO is multiplied by n. This is useful if the value of the default value is not readily available. Examples: RELTHR=10 will tend to reduce the number of atoms in a LMO. This will reduce the storage requirement and make the iterations run faster, but might make the calculation too imprecise, preventing the SCF test from being passed, and causing the run to fail. Similarly, if the results are not precise enough, then RELTHR=0.1 would tend to increase the number of atoms in a LMO. This will make the calculation become more like a MOPAC calculation, but the time required will increase. For rough work--for example, the start of a geometry optimization--then RELTHR=10 could be used. For high-precision work, either increase the precision by using RELSCF=0.1 or, preferably, do not use MOZYME. See also THRESH.
